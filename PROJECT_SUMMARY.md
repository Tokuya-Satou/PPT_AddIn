# PowerPoint Drag & Drop Add-In Project Summary

このプロジェクトは、PowerPointのスライドショー中に図形を滑らかにドラッグ＆ドロップし、かつ自由なペン描画を可能にするVSTOアドインの開発プロジェクトです。

## 1. プロジェクトの目的
標準のPowerPointやVBAでは実現が難しい「スライドショー実行中の滑らかで直感的なオブジェクト操作」を、WPFオーバーレイ技術を用いて実現します。

## 2. 現在実装済みの主要機能 (v2.0)

### 2-1. オブジェクト・ドラッグ機能
- **滑らかな移動**: PowerPointの図形を直接動かすのではなく、スライドショーの上に透明なWPFウィンドウを重ね、そこで図形の画像を100fps級のフレームレートで動かすことで「吸い付くような」操作感を実現。
- **ドラッグ対象指定**: 図形名を `Drag_` で始める（例: `Drag_Card1`）だけで自動的にドラッグ可能になります。
- **自動位置リセット**: スライドショー終了時に、動かした図形を自動的に元の位置に戻します。

### 2-2. ペン描画機能 ( InkCanvas 方式 )
- **フローティングパレット**: スライドショー中に自由に移動・最小化できるペンパレットを提供。
- **スライド別保持**: 描いた内容はスライドごとに自動保存され、ページを戻っても描画が維持されます。
- **透明オーバーレイ**: PowerPointの描画機能の制約を回避するため、透明なWPFレイヤー上で描画を行います。

### 2-3. 直近の重要な変更 (2026-04-17)
- **タッチ操作の制限**: タッチパネルでのドラッグ時に発生していたデッドロック（フリーズ）を回避するため、現在は**タッチでのオブジェクト移動を一時的に無効化**しています（マウス操作は全機能有効）。
- **Git ブランチ管理の導入**:
    - `main`: 常に動作が保証される「安定版」。
    - `dev`: 新機能開発や実験を行う「開発用」。

---

## 3. 技術スタック
- **言語/フレームワーク**: C# (.NET Framework 4.7.2), VSTO
- **UI**: WPF (Windows Presentation Foundation)
- **Hooks**: Win32 Low-Level Mouse Hook (`WH_MOUSE_LL`), Gesture Blocker (`WH_GETMESSAGE`)
- **座標同期**: スライド(4:3/16:9)と画面ピクセルをDPIスケールを考慮して動的に計算。

## 4. 今後の開発ロードマップ (Next Steps)

### 4-1. リボンUIによる制御の統合
- **Add-in On/Off**: リボンにスイッチを配置し、必要のないプレゼンではオーバーレイを起動させないようにする。
- **自動アクティブ化**: スライド内に `Drag_` 図形がある場合のみアドインを有効にするロジックの導入。

### 4-2. ペンパレットの利便性向上
- **リボンカラーギャラリー**: リボンから Office 標準のUIでペンの色や太さを自由に変更できるようにする。
- **ペンプロファイル**: よく使う設定を「ペン1」「ペン2」として登録し、パレットからワンタッチで呼び出す。

### 4-3. タッチ操作の根本解決 (デッドロック解消)
