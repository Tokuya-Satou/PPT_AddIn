## 2. スライドごとの描画保持

### 問題
スライドを移動すると描画がクリアされていた。

### 実装
`Dictionary<int, StrokeCollection> _slideStrokes` でスライドインデックスごとに保存・復元。

```
SlideShowNextSlide イベント
  → 現在スライドの Strokes を保存
  → 新スライドの Strokes を復元（なければ空）
```

`OverlayWindow` に追加したメソッド:
- `GetStrokes()` → `DrawingCanvas.Strokes.Clone()`
- `SetStrokes(StrokeCollection)` → ストロークを差し替え

スライドショー終了・開始時にディクショナリをクリア。

**変更ファイル:** `OverlayWindow.xaml.cs`, `ThisAddIn.cs`

## 3. バグ修正: 描画モード中にパレットを操作できない

### 原因
描画モード時、`InkCanvas.Background = Color.FromArgb(1,0,0,0)`（alpha=1）がスクリーン全体をカバー。OS のヒットテストでオーバーレイが優先され、パレットウィンドウにクリックが届かない。カーソルもペンのまま。

`PreviewMouseDown` で `e.Handled = true` にしても「描画しない」だけでクリックはパレットに届かないため、この方法では解決できない。

### 解決策: WM_NCHITTEST → HTTRANSPARENT
`OverlayWindow.OnSourceInitialized` で `WndProc` をフック。パレット領域への `WM_NCHITTEST` に `HTTRANSPARENT` を返す。

```csharp
if (msg == WM_NCHITTEST && _isInDrawMode && IsScreenPointOverPalette(x, y))
{
    handled = true;
    return new IntPtr(HTTRANSPARENT);
}
```

`HTTRANSPARENT` = OS がこのウィンドウを「存在しない」扱いにして、下のウィンドウ（パレット）に再ヒットテスト。カーソル表示もパレット側の矢印になる。

`_isInDrawMode` フラグを `SetPenMode`/`SetEraserMode` で `true`、`SetArrowMode` で `false` にセット。

**変更ファイル:** `OverlayWindow.xaml.cs`

## 4. バグ修正: ドラッグ&ドロップが動作しない

### 原因
マウスフックのコールバック（非WPFスレッド）から `Dispatcher.Invoke` なしで WPF オブジェクトを直接操作していた。スレッド違反の例外が `catch` に握りつぶされてサイレント失敗。

### 修正箇所（ThisAddIn.cs）

| 箇所 | 修正前 | 修正後 |
|---|---|---|
| `MouseHook_MouseDown` | `_overlayWindow.ShowSnapshot(...)` 直接呼び出し | `Dispatcher.Invoke` でラップ |
| `MouseHook_MouseMove` | `_overlayWindow.UpdatePosition(...)` 直接呼び出し | `Dispatcher.BeginInvoke` でラップ |
| `MouseHook_MouseUp` | `_overlayWindow.HideSnapshot()` 直接呼び出し | `Dispatcher.Invoke` でラップ |

COM 操作（`shape.Visible = msoFalse`）は WPF 操作より先に実行するよう順序を変更。

### ドラッグ動作の前提条件
- 図形名を PowerPoint 上で `Drag_` で始まるよう設定（例: `Drag_Card1`）
- パレットをカーソルモード（↖）にすること（ペンモード中はドラッグ無効）

**変更ファイル:** `ThisAddIn.cs`

## 補足: VSTO アドインがインストールなしで動く理由

Visual Studio でデバッグ実行（F5）すると、VS が自動でレジストリにアドインを登録する（`HKCU\Software\Microsoft\Office\PowerPoint\Addins\PPTDragDropAddIn`）。デバッグ終了後も登録は残るため、PowerPoint を単独起動してもアドインが読み込まれる。PowerPoint のCOMアドイン設定から無効化・削除可能。
